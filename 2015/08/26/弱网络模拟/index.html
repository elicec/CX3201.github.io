弱网络模拟

  • iOS
iOS开发随笔

问题引入

开发过程中有时候会遇到需要调试弱网络异常的场景，比如超时重试、转菊花等。虽然可以通过修改代码逻辑来触发场景，但毕竟不是正式流程，而且每次都要重新编译不太方便。这时候就需要一些模拟弱网络环境的手段。

解决方案

真机

如果使用真机调试来得到弱网络环境，我们可以用2G、可以跑到信号差的地方、可以建一个无法连接Internet的WiFi，分分钟搞定，so easy。当然这些方法是没有什么问题，但真这么做就太naive了。除非是测试链路，否则这样调试非累死不可。

正确的使用方式是：

开发工具中的Network Link Conditioner可以模拟出不同的网络，还可以自定义网络环境，包括带宽、延时、丢包率、协议等等，十分强大。

模拟器

模拟器的系统设置中虽然也有开发者工具箱，但是却没有包含Network Link Conditioner，所以必须想其它的办法。由于模拟器与Mac共用网络，我们可以从Mac的网络着手。

以前我曾经采用过修改Mac系统防火墙的规则实现网络限速的方法：

sudo ipfw pipe 1 config bw 350kbit/s plr 0.05 delay 10000ms
sudo ipfw add pipe 1 dst-port http

但这种方式用起来有些麻烦，而且ipfw命令在新的OS X上已经不再支持。

经过搜索发现Mac系统也提供了Network Link Conditioner这个开发工具。从xcode的菜单进入开发工具中心，下载Hardware IO Tools for Xcode，运行其中的Network Link Conditioner就能在系统设置页面增加一个选项卡：

功能和iOS的Network Link Conditioner类似，使用方法一目了然。

本文只是给出了一些比较便捷的方案，还有一些其他的方法也能达到类似或者更强大的效果，比如利用Charles、Wireshark等工具，不再详细介绍。
